# InsightReel 项目进度与后续计划

更新日期：2026-06-02  
当前分支：`main`  
当前阶段：MVP 已成型，重点进入稳定性、可观测性、插件落地与发布准备阶段。

## 1. 项目定位

InsightReel 是一个隐私优先的视频导入与 AI 分析工具，面向本地视频、在线视频字幕与网页视频分析场景，提供字幕生成、翻译、总结、关键点提取、问答和笔记能力。项目采用 React 19、Vite、TypeScript 构建，支持 IndexedDB 本地存储，并可选接入 Supabase 云同步。

## 2. 已完成进度

### 2.1 视频导入与管理

- 已支持本地视频文件导入、文件夹导入与字幕自动匹配。
- 已支持视频列表、侧边栏展示、单个删除、批量删除、选择模式和拖拽重排。
- 已抽离 `VideoDetail` 等主要展示组件，并对部分核心组件引入懒加载，降低启动负担。

### 2.2 字幕生成与处理

- 已支持 SRT、VTT 字幕导入、解析与展示。
- 已支持 Gemini、Deepgram、Whisper 相关字幕处理能力。
- 已支持 Deepgram speaker diarization，用于区分不同说话人。
- 已支持 YouTube captions 导入，提升在线视频字幕获取能力。
- 已实现字幕翻译、双语展示、字幕过滤和中文空格问题修复等优化。

### 2.3 长视频与大文件处理

- 已引入 FFmpeg.wasm 方案，支持浏览器端视频分段。
- 已实现长视频分段处理、并行字幕生成、时间戳合并和失败降级。
- 已提供 FFmpeg CDN 配置、分段处理快速开始、大文件处理与调试指南。

### 2.4 AI 分析与交互能力

- 已支持视频摘要、关键点提取、主题分析、聊天问答和笔记等核心分析场景。
- 已支持多 Provider 配置，包括 Gemini、OpenAI、Poe、Custom Provider 等方向。
- 已沉淀多 Provider 使用说明、代理配置和 Vercel 部署方案。

### 2.5 账号、认证与云同步

- 已支持 Supabase 认证与云同步配置。
- 已接入 Linux.do OAuth，并完成 callback service、redirect_uri 管理、CORS 处理、RLS 策略等多轮修复。
- 已支持同步视频元数据、字幕、分析结果、笔记和聊天记录等非视频原文件数据。

### 2.6 浏览器插件

- 已建立 Chrome Extension Manifest V3 插件目录。
- 已具备网页视频检测、content script、background service worker、popup UI 和插件构建流程。
- 已修复插件消息链路、HTML transform 与页面内容保留相关问题。

## 4. 当前风险与待补齐项

| 优先级 | 风险/问题 | 影响 | 建议处理 |
| --- | --- | --- | --- |
| P0 | `package.json` 暂未配置测试、lint、typecheck 脚本 | 回归风险高，发布前缺少自动化质量门禁 | 补齐测试与 CI，先覆盖核心服务和关键用户流 |
| P0 | YouTube captions 依赖页面结构和 caption track 可用性 | 可能因访问限制或页面变化失败 | 增加失败兜底、错误提示和回归用例 |
| P0 | 长视频分段依赖 FFmpeg.wasm 与浏览器资源 | 大文件场景可能内存占用高或加载失败 | 建立不同大小样本的基准测试与降级策略 |
| P1 | 云同步冲突与离线队列策略需要进一步明确 | 多端使用时可能出现覆盖或状态不一致 | 明确冲突合并规则、重试策略和用户提示 |
| P1 | 浏览器插件仍需完整发布链路验证 | 插件检测、权限、构建和安装流程可能存在边界问题 | 补齐插件端到端测试与打包说明 |
| P1 | 文档分散在 `docs/` 与 `@docs/` | 后续维护成本升高 | 统一文档索引和归档策略 |
| P2 | 部分架构方案仍停留在设计阶段 | 无声视频帧级分析、任务编排等能力尚未产品化 | 按里程碑拆分为可交付小版本 |

## 5. 后续计划

### 阶段一：质量基线与发布门禁

目标：确保现有能力可稳定迭代和发布。

- 增加 `typecheck`、`lint`、`test` 等脚本。
- 建立 GitHub Actions 或等效 CI 流程。
- 为字幕解析、字幕合并、长视频分段、同步服务和 YouTube captions 导入补齐基础测试。
- 增加关键错误场景的手动验证清单，包括 API key 缺失、Provider 失败、FFmpeg 加载失败、网络中断和同步冲突。

### 阶段二：字幕与长视频体验增强

目标：提升字幕生成成功率、长视频处理稳定性和用户可理解性。

- 完善 Deepgram、Gemini、OpenAI 等 Provider 的错误分类与用户提示。
- 为 YouTube captions 导入增加更明确的失败原因、重试入口和手动字幕上传兜底。
- 增加长视频处理进度、分段状态、部分结果保存和恢复提示。
- 建立长视频样本基准，记录处理耗时、内存占用、失败率和字幕完整度。

### 阶段三：云同步与账号体系完善

目标：让多端数据同步更可靠、更透明。

- 明确同步数据边界：默认同步元数据、字幕、分析结果、笔记和聊天记录，不同步原始视频文件。
- 完善离线写入队列、重试、冲突检测和冲突解决 UI。
- 增加同步状态详情页或日志面板，便于用户定位失败原因。
- 继续收敛 Linux.do OAuth 配置路径，减少 redirect_uri、client_secret、RLS 等配置错误。

### 阶段四：浏览器插件落地

目标：形成可安装、可验证、可发布的插件版本。

- 验证插件在主流视频网站、普通 HTML5 视频页面和动态加载页面中的视频检测能力。
- 完善 content script、background、popup 与主应用之间的消息协议。
- 建立插件构建、安装、升级和回归测试流程。
- 明确插件权限申请范围，减少不必要权限，降低审核风险。

### 阶段五：AI 分析能力产品化

目标：从单次分析扩展到可复用的知识沉淀。

- 优化摘要、关键点、主题、问答和笔记之间的数据关联。
- 增加分析结果的重新生成、版本对比和导出能力。
- 评估无声视频帧级分析方案的最小可行版本，优先实现音轨检测、帧采样、OCR 和视觉摘要兜底。
- 为分析结果增加结构化标签，便于后续搜索、聚合和知识库联动。
